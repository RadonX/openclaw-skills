# Frontmatter rules (do not invent blindly)

## Rule 0
Always sample existing notes in the target folder before writing.

## Baseline schemas (typical)

### Project (task-as-a-file)
---
title: "..."
status: backlog
prio: P2
created: YYYY-MM-DD
due:
blocked_by:
related_docs:
tags:
  - ...
---

### Area (evergreen)
---
title: "..."
status: evergreen
created: YYYY-MM-DD
tags:
  - ...
---

### Research
---
title: "..."
status: evergreen
created: YYYY-MM-DD
tags:
  - ...
---

## Taxonomy evolution
If introducing a new tag/field:
- add `Taxonomy decision:` section in the note
- append one entry to the folder README "Taxonomy Registry"
