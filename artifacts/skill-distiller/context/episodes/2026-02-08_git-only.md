## Episode Summary

The skill-distiller meta-skill was created in three commits starting 2026-02-08. It codifies the "distill from history" philosophy: analyze a successful conversation (initial attempt → user corrections → final accepted state), abstract the underlying principles, and synthesize a new reusable skill. The initial commit captured core principles from the handoff skill creation process. A subsequent commit rewrote the skill with progressive disclosure (lean SKILL.md router + focused references/), multi-source history support, proposal-first safety, and archived the prior Gemini draft. A final commit added an installer-facing README.

## Key Decisions / Constraints

- SKILL.md must be a lean router (≤100 lines), not a monolith — details live in references/
- Skill interface uses subcommands over flags for mode switching; flags are for parameters only
- Proposal-first: full proposed file tree + contents must be printed before any file writes
- File writes gated by explicit `--apply` flag or user confirmation — no silent mutations
- Multi-source history: `--source context|session|paste` for flexibility in how conversation history is obtained
- Prior Gemini-generated iteration preserved as `skill-distiller-gemini` rather than deleted, for comparison
- Installer README recommends symlink-per-skill pattern (`~/.openclaw/skills/<name> -> ~/repo/config/openclaw/skills/<name>`)

## Key User Quotes

1. "I do not like the skill you just created. too specific. try using skill-distiller-gemini/SKILL.md to create the skill again." — user rejects distiller output as overly specific, switches to gemini variant
2. "do we need so many args for this skill though?" — user critiques the shape of distilled SKILL.md output for excessive arguments
3. "not from short input, but from the session!!" — user corrects distiller to extract from full session history, not a brief summary
4. "still to many args" — repeated rejection of distiller's verbose output shape
5. "话说 skill-distiller 或 skill-distiller-gemini 是不是有点问题，为啥这个 intent-writer skill create起来这么困难" — user questions whether the distiller itself is fundamentally flawed when a simple skill is hard to produce

## Relation to skill evolution

This is a clear evolution episode: the skill-distiller was created from scratch across three commits, each refining its structure and interface. The commits show iterative design — starting with a monolithic SKILL.md + principles, then refactoring into progressive disclosure with separated concerns (process, output contract, safety, help). The archival of the Gemini draft and addition of installer docs further demonstrate intentional skill architecture decisions. No prior version of skill-distiller existed; this is the initial creation arc. The Feb 14 session evidence confirms a persistent gap: user consistently found distiller output too verbose and too specific, suggesting the skill needs further refinement toward minimal, session-driven distillation.
