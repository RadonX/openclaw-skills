## Episode Summary

The skill-distiller meta-skill was created in three commits starting 2026-02-08. It codifies the "distill from history" philosophy: analyze a successful conversation (initial attempt → user corrections → final accepted state), abstract the underlying principles, and synthesize a new reusable skill. The initial commit captured core principles from the handoff skill creation process. A subsequent commit rewrote the skill with progressive disclosure (lean SKILL.md router + focused references/), multi-source history support, proposal-first safety, and archived the prior Gemini draft. A final commit added an installer-facing README.

## Key Decisions / Constraints

- SKILL.md must be a lean router (≤100 lines), not a monolith — details live in references/
- Skill interface uses subcommands over flags for mode switching; flags are for parameters only
- Proposal-first: full proposed file tree + contents must be printed before any file writes
- File writes gated by explicit `--apply` flag or user confirmation — no silent mutations
- Multi-source history: `--source context|session|paste` for flexibility in how conversation history is obtained
- Prior Gemini-generated iteration preserved as `skill-distiller-gemini` rather than deleted, for comparison
- Installer README recommends symlink-per-skill pattern (`~/.openclaw/skills/<name> -> ~/repo/config/openclaw/skills/<name>`)
