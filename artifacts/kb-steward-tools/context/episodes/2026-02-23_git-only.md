## Episode Summary

The kb-steward-companion skill was renamed to kb-steward-tools to better reflect its role as a query/maintenance toolset rather than a companion. A major directory structure v2.0 was then introduced, adding layered project organization (active/completed/backlog with emoji-prefixed folders). After adding a local DIRECTORY_STRUCTURE_V2.md reference, the docs were deduplicated by extracting a core spec shared with kb-steward and creating a tools-specific extension doc. Finally, the "v2" version suffixes were dropped from filenames as the structure became the current standard.

## Key Decisions / Constraints

- Skill renamed from "kb-steward-companion" to "kb-steward-tools" — new folder with fresh SKILL.md, all scripts ported
- v2 directory structure adds project-level folders with PROJECT.md, tasks/, references/ subdirectories
- `create-project.sh` and `create-task.sh` scripts added to support new structure
- `--scope active|completed|backlog` flag added to todo.sh and prio.sh for filtering
- Directory structure docs deduplicated: single core spec at `../kb-steward/references/DIRECTORY_STRUCTURE.md`, tools-specific behavior in `TOOLS_EXTENSION.md`
- "v2" suffix removed from all filenames once the structure became canonical — version suffixes create maintenance burden

## Relation to skill evolution

This is `evolution` — the skill's folder name, interface (new scripts, new flags), and reference documents were all modified across these commits. The rename itself is a structural change to the skill's identity, and the v2 directory support changes its operational contract (new commands, new vault layout expectations). The subsequent deduplication and v2-suffix removal are refinement passes that finalize the shape of the skill's documentation.
