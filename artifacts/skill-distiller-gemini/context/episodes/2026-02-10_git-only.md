## Episode Summary

While building the new skill-distiller (Claude-based), the previous Gemini-generated version was preserved as `skill-distiller-gemini`. This commit archives the Gemini draft — its SKILL.md (a 6-step workflow from conversation analysis to skill synthesis) and PRINCIPLES.md (5 distilled skill-design rules) — so the two approaches can be compared and referenced. The Gemini variant uses a single command with optional `--session`, `--message-id-start/end`, and `--apply` flags rather than the subcommand-based architecture of the newer skill-distiller.

## Key Decisions / Constraints

- Gemini draft archived under `-gemini` suffix rather than discarded, enabling future comparison of design approaches
- The Gemini version's PRINCIPLES.md (progressive disclosure, CLI-like interface, safety-first, self-containment) informed but did not dictate the new skill-distiller's design
- The new skill-distiller adopted a progressive-disclosure / lean-router architecture that the Gemini draft also prescribed but implemented differently

## Relation to skill evolution

This is a clear `evolution` episode: the commit explicitly introduces the skill-distiller-gemini skill directory as an artifact of the skill-distiller's own development. The Gemini iteration's approach — single-command with flags, Root Cause Analysis workflow — was contrasted against the new subcommand-based architecture, and the old version was preserved as a reference point rather than deleted.
