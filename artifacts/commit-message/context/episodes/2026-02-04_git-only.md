## Episode Summary

The commit-message skill was created from scratch as a reusable instruction module for drafting high-quality Conventional Commit messages. The skill wraps a CLI-like interface that inspects staged/unstaged diffs and recent chat context to produce intent-driven commit narratives. Two files were introduced: SKILL.md (the main instruction + workflow) and references/NARRATIVE.md (the style reference for structuring subject lines and bodies).

## Key Decisions / Constraints

- Commit messages must explain **intent + impact** (why), not just list changed files (what)
- Proportionality principle: small diff → short message, large refactor → structured body with H4 groupings
- CLI-style invocation with explicit flags: `--apply` (write), `--amend` (modify last), `--repo` (target directory)
- Safety hard line: never run `git push`; no repo modification without `--apply`
- Conversation context may inform the "why" but must not claim changes absent from the staged diff
