## Episode Summary

The user referenced their ~/repo palimpsest repo as having good metadata tracking and wanted a similar system for OpenClaw skills. This triggered the design and implementation of a per-skill metadata infrastructure: `meta.yaml` (contract + provenance), `decisions.yaml` (append-only decision log), and context notes under `artifacts/<skill>/context/episodes/`. The session also covered kb-steward's own evolution (kbRoot default change, docs additions) and the creation of a subagent-driven episode extraction pipeline to retroactively trace skill evolution from session logs and git history.

## Key Decisions / Constraints

- **YAML as primary format** for metadata files (user preference)
- **Decision log records only important changes**, not changelog noise
- **`meta.yaml` description is routing-only**; behavioral guardrails stay in SKILL.md/refs
- **`message_id` is not durable** (session JSONL gets deleted); shift to artifact-backed evidence with excerpts and git references
- **Separate episodes per session**, not one unified context note — to track version evolution over time
- **Subagents for session reading** to conserve context; default to GLM model
- **Session logs + git history** both used as evidence sources for retrospective tracing
- **Owner: RadonX; primary model: gemini 2.5 pro** for skill generation
