## Episode Summary

Eight commits on 2026-02-04 trace the full birth and maturation of the `handoff` skill within a single day. Starting from a monolithic SKILL.md that mixed temporary handoffs with permanent doc updates, the skill was progressively refactored into a subcommand-first CLI with a lean router file and on-demand reference loading. The final shape: a smart-default command that auto-decides new vs update, a `load` subcommand that primes session context by reading handoffs and linked docs, and a `know` subcommand for guarded knowledge-base maintenance.

## Key Decisions / Constraints

- **Paths must be portable**: absolute `/Users/ruonan/...` paths replaced with `~/` prefix throughout (commit 7e4f94a8).
- **Smart default over explicit flags**: `--new/--update` flags removed; agent decides new vs update deterministically, only asking on ambiguity (commit bc5692f7).
- **Progressive disclosure**: heavy workflow logic extracted from SKILL.md into `references/` to reduce token burn on every invocation (commit 864c5d35).
- **Load = context primer, not just navigation**: upgraded from "find the right file" to "read the handoff + all linked docs and prime the session" (commit 71e78db3).
- **Knowledge writes are guarded**: `know` subcommand defaults to propose-first; requires `--apply` + explicit user confirmation.
- **Help surface added**: `/handoff help` subcommand + human-facing README.md for discoverability (commit 905f0e57).
