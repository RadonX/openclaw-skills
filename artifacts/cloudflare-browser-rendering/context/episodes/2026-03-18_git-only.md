## Episode Summary

The cloudflare-browser-rendering skill was created to fetch fully JS-rendered web pages via Cloudflare's headless Chromium API — primarily targeting WeChat Official Account articles and other JS-heavy SPAs. The skill includes a Python helper script, API parameter reference, and credential setup from SECRETS.md. A follow-up commit immediately changed the default endpoint from `/content` to `/markdown`, reasoning that markdown output is cleaner and requires no HTML parsing.

## Key Decisions / Constraints

- Credentials sourced from `SECRETS.md` (`CLOUDFLARE_ACCOUNT_ID`, `CLOUDFLARE_BR_TOKEN`) rather than environment variables alone
- Default endpoint changed from `/content` (returns HTML, needs parsing) to `/markdown` (returns clean markdown directly)
- CLI flag flipped: `--markdown` became `--content` (opt-in to the heavier HTML path)
- `waitFor` parameter explicitly documented as unsupported (causes 400 errors)
- `rejectResourceTypes` recommended for faster text-only fetches
