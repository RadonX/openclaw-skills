## Episode Summary

The bindings-simplifier skill was created from scratch as a new OpenClaw skill module. It provides a structured approach to simplifying OpenClaw bindings configurations by eliminating redundancy while preserving functionality. The skill includes a SKILL.md with three core strategies (account-level bindings, topic-level merging, exception preservation) and four reference documents covering auditing, examples, help, and prioritization.

## Key Decisions / Constraints

- Simplification triggered when bindings exceed ~30 entries or show repetitive per-topic patterns
- Three core strategies in priority order: account-level consolidation, topic-level merging, exception preservation
- Dry-run mode (`--dry-run`) required before any write operations
- Account-level simplification only safe when accountId maps to a single agent
- Topic merging requires verifying consistent `requireMention` and `allowFrom` settings across topics
- Explicit exception topics must be preserved when they need special handling

## Relation to skill evolution

This is an `evolution` episode — the entire skill was introduced in a single commit, establishing its interface contract (`/bindings-simplifier [--dry-run] [--show-changes]`), simplification strategies, and safety constraints. No prior version existed; this is the genesis of the skill.
